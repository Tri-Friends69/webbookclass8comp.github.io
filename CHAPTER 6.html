Programming Languages and Language Translators

Absence of language makes communication difficult.
Let's now try to communicate with a computer which cannot do anything on its own and quires instructions. Do you think you can give instructions to computer in your mother tongue Which language computers can understand?

Computer can understand only one language and it is called machine language. It consists of series of 0's and 1's.

Programming Languages

You can well imagine, writing in 1s and Os al day will leave you prone to mistakes. By using codes resembling English, programming becomes much easier.The usage of mnemonic codes such as LDA for load, STA for store and SUB for subtract means the code is easier to read and write. The programming language that uses memonics for writing codes is called assembly language.With the need for making programming simpler, High Level Languages were developed These languages make use of English words and mathematical symbols for writing programs Basic is one of the High Level Languages (HLL) used for giving instructions to the computer.

Early computers were programmed in machine languages only.

Assembler

Assembler converts an assembly language program into machine code which is direc understood by computer.

High Level Language Translator

Since there are different High Level Languages, hence we need different translators for different languages. If we are working on C++ language, we will require C++ translator (compiler), and if we are working on QBasic then we require translator(interpreter) for QBASIC. This translation is done in 2 ways. Interpreter

The Interpreter fetches single instruction or a single step of program written in HLL, translates it into machine code and executes it immediately.
This cycle of fetch to translate to execute is repeated for all the instructions. If there is an error in the instruction, the interpreter terminates its process at that statement and displays an error message. After correcting the error, the program in HLL is submitted again. QBASIC is one of the examples of interpreter run languages.

Compiler

Language processor that translates the complete program as a whole in one go into machine code is called compiler.

The program written in HLL is called Source Code and the program translated into machine code is called the Object Code.


Compilation process is successful if source code is free of errors. If there are any errors in the source code, the compiler specifies all the errors with line numbers. The errors must be removed before the compiler can successfully recompile the source code again.
